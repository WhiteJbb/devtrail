# Vault-AI 연동 설계

작성일: 2026-06-25  
문제: 상용 AI(Claude Code, ChatGPT, Gemini 등)가 vault 내용을 세션마다 모른다.

---

## 1. 문제 정의

### 1a. 경로 이식성

vault 경로는 머신마다 다르다.

| 머신 | 경로 |
|---|---|
| 개발 PC | `C:\Users\admin\Desktop\personal-vault` |
| 노트북 | `~/Documents/personal-vault` 등 |

CLAUDE.md에 절대 경로를 박으면 다른 컴퓨터에서 즉시 깨진다.  
현재 CLAUDE.md의 `{VAULT}` 는 치환되지 않는 플레이스홀더라 실효 없음.

### 1b. 실제 파일 위치 불일치

CLAUDE.md가 참조하는 경로: `{VAULT}/40_AgentMemory/Core/`  
실제 파일 위치: `40_AgentMemory/00_Profile.md` ~ `05_OpenLoops.md` (Core/ 서브폴더 없음)

### 1c. AI별 접근 방법이 없음

| AI | 현재 상황 |
|---|---|
| Claude Code | CLAUDE.md 규칙은 있지만 경로 깨짐 → 읽지 못함 |
| ChatGPT / Gemini 웹 | vault 파일 자체에 접근 불가 |
| Telegram 봇 | `AgentMemoryLoader` 코드는 있음. 동작은 하지만 세션 시작 시 자동 주입 없음 |

---

## 2. 핵심 아이디어 — 파생 파일 전략

vault 경로는 `.env`가 알고 있다. work-agent도 `.env`를 읽는다.  
→ **work-agent가 vault를 읽어 프로젝트 내 고정 경로에 브리핑 파일을 생성**한다.

```
vault (머신마다 다른 경로)
    ↓ work-agent briefing
.claude/briefing.md  ← 항상 같은 상대 경로
    ↓
Claude Code / ChatGPT (붙여넣기) / Telegram 봇
```

`.claude/briefing.md` 는:
- 프로젝트 루트 기준 고정 경로 → CLAUDE.md에 경로 하드코딩 가능
- gitignore 처리 (머신별 내용, 커밋 불필요)
- AI가 읽기 좋은 구조화된 마크다운

---

## 3. `work-agent briefing` 명령 설계

### 3a. 입력

`AgentMemoryLoader.load()` 가 읽는 파일 6개 + active tasks:

| 파일 | 내용 |
|---|---|
| `40_AgentMemory/00_Profile.md` | 기본 정체성, 스킬, 작업 방식 |
| `40_AgentMemory/01_CurrentFocus.md` | 현재 최우선 프로젝트 |
| `40_AgentMemory/02_ProjectMap.md` | 전체 프로젝트 지도 |
| `40_AgentMemory/03_WritingStyle.md` | 글쓰기 스타일 가이드 |
| `40_AgentMemory/04_CareerContext.md` | 커리어 목표, 지원 직무 |
| `40_AgentMemory/05_OpenLoops.md` | 미해결 이슈 목록 |
| TaskAgent.list_active() | 현재 활성 태스크 |

### 3b. 출력 — `.claude/briefing.md`

```markdown
<!-- generated by work-agent briefing — do not edit manually -->
<!-- updated: 2026-06-25T14:30:00 -->

## [Profile]
AI 서비스를 기획·구현하는 개발자 지망생. AI Engineer / FDE 직무 준비 중.
스택: Python, FastAPI, RAG, Qdrant, React, Docker.

## [CurrentFocus]
1. work-agent — E2E 루프 안정화 (capture → distill → promote → output)
2. XCoreChat — 사내 규정 RAG, 포트폴리오 정리 단계
3. Orbit — 브라우저 에이전트, Chrome Extension MVP
4. 취업 준비 — LG CNS AX / Dalpha / Toss 목표

## [ActiveTasks]
· #3 Telegram LLM 자연어 연동 설계 (이번 주)
· #4 weekly-distill E2E 테스트 (언제든지)

## [OpenLoops]
- work-agent: LONG_WRITER_PROVIDER 환경변수 미연결
- work-agent: _FallbackProvider vs FallbackChain 중복 제거 미완료
- Orbit: Chrome Extension 기본 구조 미착수

## [KeyRules]
- source에 없는 수치·성과는 만들지 않는다
- 기능 추가보다 MVP 루프 안정화 우선
- AI가 만든 내용은 검토 후 사용
```

길이 목표: **1500~2500자** (어떤 AI든 컨텍스트에 부담 없는 크기)

### 3c. 구현 위치

```
app/agents/briefing_agent.py    ← 새 파일
app/cli.py                      ← briefing 명령 추가
```

`BriefingAgent.generate()`:
1. `AgentMemoryLoader(vault_dir).load()` → 섹션별 요약 추출
2. `TaskAgent().service.list_active()` → 활성 태스크
3. 마크다운 조합 → `.claude/briefing.md` 덮어쓰기
4. 생성 타임스탬프 주석 삽입

## 4. Claude Code 연동

### 4a. CLAUDE.md 수정

현재 (깨진 상태):
```
- `{VAULT}/40_AgentMemory/Core/` — 프로젝트별 핵심 컨텍스트 (가장 먼저 읽을 것)
- `{VAULT}/40_AgentMemory/05_OpenLoops.md` — 미해결 이슈 목록
```

변경 후:
```
- `.claude/briefing.md` — 세션 컨텍스트 요약 (없으면 `work-agent briefing` 실행 후 읽을 것)
```

경로 변경 없음, vault 경로 불필요.

### 4b. 세션 시작 hook (선택)

`.claude/settings.json` 에 hook 추가:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "work-agent briefing"
          }
        ]
      }
    ]
  }
}
```

hook이 있으면 세션 최초 도구 사용 전 briefing을 자동 갱신한다.  
hook 없이도 수동으로 `work-agent briefing` 실행 후 세션 시작하면 동일 효과.

**권장 운영 방식**: 매일 작업 시작 전 한 번 실행. hook은 선택 사항.

## 5. ChatGPT / Gemini 연동 (붙여넣기 방식)

vault 파일 접근이 불가능하므로 내용을 텍스트로 전달해야 한다.

### 5a. 파일 복사 방식

```bash
work-agent briefing          # .claude/briefing.md 갱신
cat .claude/briefing.md | clip   # 클립보드 복사 (Windows)
```

ChatGPT/Gemini 세션 첫 메시지에 붙여넣기.

### 5b. Telegram `/briefing` 명령 (보조)

모바일에서 컨텍스트가 필요할 때:

```
/briefing
```

→ `.claude/briefing.md` 내용을 Telegram 메시지로 전송.  
붙여넣기 없이 모바일에서 현재 컨텍스트 확인 가능.

`CommandRouter._dispatch()` 에 `cmd == "briefing"` 분기 추가.

## 6. Telegram 봇 자동 주입 (Part 1 설계와 연결)

[new-command-design.md](./new-command-design.md) 의 시스템 프롬프트 구조와 연동:

- `_build_system_prompt()` 에서 `AgentMemoryLoader.load()` 를 직접 호출하는 대신  
  `.claude/briefing.md` 를 읽어 system prompt 로 주입하는 방식도 가능
- 장점: Claude Code와 Telegram 봇이 동일한 브리핑 소스 사용
- 단점: `.claude/briefing.md` 가 갱신 안 된 경우 stale 컨텍스트

**결정 보류**: `AgentMemoryLoader` 직접 호출(항상 최신) vs `.claude/briefing.md` 읽기(소스 통일)

## 7. 갱신 전략

| 트리거 | 갱신 방법 |
|---|---|
| 작업 시작 전 (수동) | `work-agent briefing` |
| git commit 시 (자동) | post-commit hook에 `work-agent briefing` 추가 |
| Claude Code 세션 시작 (자동) | settings.json `PreToolUse` hook |
| `capture-session` 실행 시 (자동) | CaptureAgent 내부에서 briefing 갱신 호출 |

권장: **post-commit hook** 에 추가. 커밋할 때마다 갱신되므로 별도 신경 쓸 필요 없음.

---

## 8. `.gitignore` 처리

`.claude/briefing.md` 는 머신별 내용이므로 gitignore:

```
# .gitignore
.claude/briefing.md
```

다른 파일(`.claude/settings.json`, `.claude/global.md`)은 그대로 커밋.

## 9. 구현 순서 (나중에 할 때 참고)

1. `app/agents/briefing_agent.py` — `BriefingAgent.generate()` 작성

5. (선택) `settings.json` — PreToolUse hook 추가
6. (선택) `CommandRouter` — `/briefing` 명령 추가
