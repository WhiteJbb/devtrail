## 지시사항

위 작업 기록을 분석해 이력서·포트폴리오에 활용할 수 있는 후보를 추출하세요.

### 반드시 지켜야 할 규칙
1. 작업 기록에 명시된 사실만 사용한다. 없는 수치·결과를 만들어내지 않는다.
2. "개선했다", "최적화했다" 등의 표현은 근거가 있을 때만 사용한다.
3. 결과가 불분명하면 "결과 확인 필요"로 표시한다.
4. 문제 → 행동 → 결과 구조로 작성한다 (결과가 없으면 결과 확인 필요).
5. 과장 금지. 실제 source에 없는 수치 금지.
6. **title은 구체적으로**: 어떤 시스템의 어떤 문제를 다뤘는지 드러나야 한다.
   "AI 기반 시스템 개선", "품질 관리" 같은 추상 명사 나열 제목은 금지.
   좋은 예: "Telegram 봇 태스크 번호 재배열 문제를 안정 ID 스냅샷으로 해결"
7. resume_bullets 문자열에 "•", "-" 같은 불릿 기호를 포함하지 않는다 —
   렌더링 단계에서 붙이므로 중복 기호가 생긴다. 문장만 쓴다.

### 반환 형식

아래 JSON 형식으로만 응답하세요:

```json
{
  "career_bullets": [
    {
      "title": "후보 제목 (예: XCoreChat 실시간 메시징 아키텍처 설계)",
      "project": "프로젝트명 (없으면 빈 문자열)",
      "source_evidence": "작업 기록에서 이 후보의 근거가 되는 내용",
      "resume_bullets": ["이력서에 바로 쓸 수 있는 문장 1 (불릿 기호 없이)", "문장 2"],
      "portfolio_description": "포트폴리오 설명 (2-3문장)",
      "interview_points": ["면접에서 설명할 포인트 1", "포인트 2"],
      "caveats": "주의사항이나 불확실한 점 (없으면 빈 문자열)",
      "source_refs": ["참조한 기록 경로"],
      "tags": ["career", "portfolio", "resume"]
    }
  ]
}
```

후보가 없으면 `{"career_bullets": []}` 를 반환합니다.
